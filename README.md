# Channel OS v0.2.1 — Analytics Pro UI

Telegram-бот для управления каналами, публикациями и аналитикой.

## Что нового в v0.2.1

В v0.2.0 был добавлен Analytics Core — сбор снимков статистики в PostgreSQL.  
В v0.2.1 добавлен пользовательский интерфейс Analytics Pro.

### Возможности

- 📊 новый раздел «Аналитика»;
- 📢 выбор канала для аналитики;
- 👥 текущие подписчики;
- 📈 рост за 24 часа;
- 📅 рост за 7 дней;
- 🗓 рост за 30 дней;
- 📄 отчёт за 24 часа;
- 📅 история последних снимков;
- ▁▂▃ мини-графики динамики.

## BotHost

BotHost запускает проект через:

```bash
node src/app.js
```

Поэтому Prisma инициализируется внутри `src/config/prisma.js`.

## Переменные окружения

```env
BOT_TOKEN=токен_бота
DATABASE_URL=postgresql://...
```

## Деплой

```bash
git add .
git commit -m "Release v0.2.1 Analytics Pro UI"
git push
```

После этого сделай Redeploy на BotHost.

## Важно

Telegram Bot API не даёт обычному боту полный список подписчиков канала.  
Поэтому Analytics Pro считает рост на основе снимков общего количества подписчиков.

## v0.2.1.1

Health Score временно убран из интерфейса. Вернём его позже, когда будет прозрачная формула расчёта на основе реальных метрик канала.
## v0.2.1.2 — Analytics Snapshot Fix

Исправлена аналитика:

- Health Score временно убран.
- Основная аналитика использует только `AnalyticsSnapshot`.
- Добавлена кнопка ручного сбора снимка.
- Если данных нет, бот объясняет причину.
- Если Telegram не отдаёт число подписчиков, ошибка показывается в интерфейсе.

Для проверки после деплоя:

1. Открой `📊 Аналитика`.
2. Нажми `🔄 Обновить снимок сейчас`.
3. Проверь, что появились подписчики и время обновления.
4. Если будет ошибка — проверь, что бот является администратором канала.
## v0.2.1.3 — Analytics Engine Debug

Добавлен режим диагностики.

### Команды

```text
/debug
/debug_analytics
```

Показывают:

- запущен ли Analytics Scheduler;
- интервал сбора снимков;
- последний запуск;
- следующий запуск;
- последний результат;
- последнюю ошибку.

### Ручная проверка

1. Открой бота.
2. Напиши `/debug`.
3. Нажми `🔄 Собрать снимки сейчас`.
4. Перейди в `📊 Аналитика`.
5. Проверь `📈 Общий обзор`.

### Настройки .env

```env
ANALYTICS_INTERVAL_MINUTES=30
ANALYTICS_INITIAL_DELAY_SECONDS=15
```

## v0.2.1.4 — Telegram API Fix

В Telegraf вызов количества участников выполняется через:

```js
telegram.callApi("getChatMemberCount", {
  chat_id: channelTelegramId
});
```

После деплоя проверь:

1. Напиши `/debug`.
2. Нажми `🔄 Собрать снимки сейчас`.
3. В результате должно быть `Успешно: 1`.
4. Открой `📊 Аналитика` → `📈 Общий обзор`.
